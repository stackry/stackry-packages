# Stackry Packages

Public package shelf for Stackry station appliance installers.

This repository is distribution-only. Product source lives in
`stackry/stackry-vision`; this repo hosts the signed APT repository used by
station operators.

## Target Install Flow

Before DNS is configured:

```bash
curl -fsSL https://stackry.github.io/stackry-packages/install.sh | sudo bash
sudo apt-get update
sudo apt-get install -y stackry-cli
```

After DNS is configured:

```bash
curl -fsSL https://packages.stackry.com/install.sh | sudo bash
sudo apt-get update
sudo apt-get install -y stackry-cli
```

## Layout

```text
install.sh
apt/
  dists/
  pool/
  stackry-packages.gpg
```

The APT metadata and `.deb` files are generated by release automation. Do not
edit generated package indexes by hand.

## Security

- The repository is public because packages contain no secrets.
- Stations trust package signatures, not GitHub login.
- `install.sh` installs the Stackry public package signing key into
  `/usr/share/keyrings/stackry-packages.gpg`.
- The APT source uses `signed-by=/usr/share/keyrings/stackry-packages.gpg`.

## Publishing

`scripts/publish-apt-repo.sh` publishes `stackry-cli_*.deb` files into the
local APT tree, regenerates package indexes, exports the public signing key,
and signs the Release metadata.

The `Publish Stackry CLI` workflow downloads package artifacts from a
`stackry/stackry-vision` workflow run, signs the APT repository, and commits the
generated repository files to `main`.

Required repository secrets:

```text
STACKRY_APT_GPG_PRIVATE_KEY
STACKRY_APT_GPG_PASSPHRASE
STACKRY_APT_GPG_KEY_ID
```
